# Практические работы по проектированию мобильных интерфейсов

> **Тема проекта:** Образовательное приложение **LearnUp**

---

## ПЗ №1 — Выбор и аналитическое исследование темы

### Цель
Сформировать понимание типов мобильных интерфейсов, осознанно выбрать тему и провести первичный анализ интерфейсных решений.

### Этапы выполнения

| Этап | Название | Что делать |
|------|----------|------------|
| 1 | Выбор тематического направления | Выбрать тему, сформулировать название и назначение, обосновать выбор |
| 2 | Определение целевой аудитории | Описать пользователя (возраст, занятия, мотивация), цели, контекст использования |
| 3 | Подбор и анализ аналогов | Найти 3-5 приложений, собрать скриншоты, проанализировать структуру/навигацию/стиль |
| 4 | Сравнительный анализ | Составить таблицу сравнения, выделить паттерны, определить плюсы/минусы |
| 5 | Формулировка выводов | Определить удачные решения, проблемные зоны, требования к своему проекту |

### Что сдавать
- **Figma-файл:** экраны, разметка, таблица сравнения, выводы
- **PDF-презентация:** 8-10 слайдов с описанием этапов

### Критерии оценки
1. Обоснованность выбора темы
2. Глубина анализа целевой аудитории
3. Полнота сбора и анализа аналогов
4. Качество сравнительного анализа
5. Конкретность и применимость выводов

---

## ПЗ №2 — Lo-Fi прототипирование и навигация

### Цель
Проектировать логическую структуру приложения и создавать прототипы низкой детализации, отражающие основные сценарии и навигацию.

### Этапы выполнения

| Этап | Название | Что делать |
|------|----------|------------|
| 1 | Уточнение пользовательского сценария | Зафиксировать тему, главную цель пользователя, описать основной и второстепенные сценарии |
| 2 | Формирование списка экранов | Составить перечень экранов, сгруппировать по блокам, определить иерархию |
| 3 | Проектирование навигации | Выбрать тип навигации (таб-бар/боковое меню/иерархия), определить переходы |
| 4 | Построение User Flow | Нарисовать схему экранов и переходов, отметить основные/альтернативные пути |
| 5 | Разработка Lo-Fi прототипов | Создать каркасные макеты без цвета, разместить элементы, обозначить иерархию |
| 6 | Связь экранов в прототип | Настроить переходы, проверить кликабельные зоны, пройти сценарий |
| 7 | Анализ и корректировка | Провести тестирование, зафиксировать ошибки, внести правки |

### Что сдавать
- **Figma-файл:** Lo-Fi экраны, User Flow схема, интерактивный прототип
- **PDF-презентация:** 8-10 слайдов с описанием этапов

### Критерии оценки
1. Соответствие структуры и навигации сценариям
2. Логичность и целостность карты переходов
3. Понятность навигационной модели
4. Качество Lo-Fi прототипов
5. Обоснованность проектных решений

## ПЗ №3 — Дизайн-система и UI-кит

### Цель
Создать целостный визуальный язык продукта в виде дизайн-системы и UI-кита, обеспечивающих единообразие интерфейса.

### Этапы выполнения

| Этап | Название | Что делать |
|------|----------|------------|
| 1 | Формирование визуальных основ | Определить характер (строгий/дружелюбный), цветовую палитру, типографику, сетку |
| 2 | Проектирование компонентов | Разработать кнопки, формы, навигацию, карточки, иконки |
| 3 | Проработка состояний | Для каждого компонента: обычное, активное, фокус, отключённое, ошибка |
| 4 | Сборка UI-кита | Оформить компоненты в Figma, настроить Variants, разложить по разделам |
| 5 | Проверка на применимость | Собрать один экран из UI-кита, проверить согласованность |
| 6 | Оформление правил | Описать назначение компонентов, принципы визуальной системы |

### Структура UI-кита

```
UI-Kit/
├── Foundations/
│   ├── Colors (Primary, Secondary, Neutral, Status)
│   ├── Typography (H1-H4, Body, Caption, Button)
│   ├── Grid (8px модуль, 16px margins)
│   └── Spacing (4, 8, 12, 16, 24, 32, 48)
├── Components/
│   ├── Buttons (Primary, Secondary, Text, Icon)
│   ├── Inputs (Text, Password, Search, Textarea)
│   ├── Cards (Course, Lesson, Achievement)
│   ├── Navigation (Tab Bar, Header, Back)
│   └── Feedback (Toast, Modal, Empty State)
└── Patterns/
    ├── Forms
    ├── Lists
    └── Onboarding
```

### Что сдавать
- **Figma-файл:** UI-кит с компонентами, пример экрана
- **PDF-презентация:** 8-10 слайдов с описанием этапов

### Критерии оценки
1. Целостность визуального языка
2. Полнота компонентов и состояний
3. Корректность применения принципов юзабилити
4. Качество UI-кита в Figma
5. Обоснованность решений

## Связь между работами

```
ПЗ №1 (Анализ)
    │
    ▼ Выводы и требования
    │
ПЗ №2 (Lo-Fi прототипы)
    │
    ▼ Структура и навигация
    │
ПЗ №3 (UI-кит и Hi-Fi)
```

Каждая следующая работа строится на результатах предыдущей. Выводы из ПЗ №1 определяют структуру в ПЗ №2, а Lo-Fi прототипы становятся основой для визуального стиля в ПЗ №3.
